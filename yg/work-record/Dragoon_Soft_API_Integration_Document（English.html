  1. Please contact Dragoon Soft to ask GAME API address, GAME API public key, and Agent Account.
  2. Please provide IP that connect to us before you get started.(Please filled up in MIF.)
  3. The integration transmit formation is Json. And is encoded by UTF-8.

Request Signing

API Data Signing Mathod

  • All Games API requests have to be signed by Operator.
  • All Wallet API requests have to be signed by Dragoon Soft
  • All Wallet API request must be practiced and verified by the Operator.

Encryption Method and Process

  • Both parties should verify all requests will be signed with RSA-SHA256.
  • Before the integration, the Operator generates a private/public key pair, and provides the public key to Dragoon Soft.
  • Dragoon Soft generates a RSA private/public key pair, and provides the public key to operator. (RSA KEY lenth should be 1024 or 2048)
  • The body of all requests will be signed with RSA-SHA256 using the respective private key and encoded to BASE64
  • Body JSON sort shouldn’t be changed.
  • The signature should be placed in the X-Ds-Signature request header.
  • The Operator needs to verify all Wallet API requests using the public key provided by Dragoon Soft.
  • Dragoon Soft verifies all Games API requests using the public key provided by the Operator.
  • The body of all requests will be signed with RSA-SHA256 using the respective private key and encoded to BASE64, the signature should be placed in the X-Ds-Signature request header.

Example

POST /v1/member/create HTTP/1.1
Content-Type: application/json
X-Ds-Signature: byMYoU1abC/86EVvEbwNtumfgZnlbGaAwbeJ3qu668RCZRnkFv8B3fCRAzM9rDnDdjommrhK0B3BVmbpdCv6UzyY1avfxE2lbihvrF/LMkY4oKOyL8IJuGBbSLgzpI+hKYzcb+qD/En9ZAH0V4gNXh6RqI/XIgfWhB55zF9pBs4=

{
  "agent": "agent01",
  "account": "player001",
  "token": "1ceda4754e7e413592a6450fb224165f"
}

Important:

Parameter should be in JSON format.

Warning:

X-Ds-Signature Signature failure will cause the inquiry failed.

Wallet API-Get Balance

To get the balance of member

Request URL

https://<url>/api/wallet/balance [POST]

Request Parameters

Parameter Type Enforce Description
agent String Yes agent account
account String Yes member account
token String Yes wallet access token

Example

{
  "agent": "agent01",
  "account": "player001",
  "token":"e278a9ea38da42dd9b84148c2f99a031"
}

Response Parameters

Parameter Type Description
balance Number member balance PS:support to second decimal place
status Number wallet message code

Example

{
  "balance": 99999.1,
  "status": 1
}

Wallet API-Bet Request

Called when the member bets.

Operator is expected to decrease member’s balance by amount and return new balance.

Request URL

https://<url>/api/wallet/bet [POST]

Request Parameters

Parameter Type Enforce Description
trans_id String Yes transaction id PS: UUID
amount Number Yes transaction amount PS:support to second decimal place
agent String Yes agent account
account String Yes member account
token String Yes wallet access token

Important:

operating transactionamount should be a positive number

Example

{
  "trans_id": "9b2bb872-332a-4d8d-87aa-2e18dc4edff8",
  "amount": 8000,
  "agent": "agent01",
  "account": "player001",
  "token": "88406af3628342f19cea6042d0cb2267"
}

Response Parameters

Parameter Type Description
trans_id String transaction id PS:equal to transaction id requested
balance Number member balance PS:support to second decimal place
status Number wallet message code

Example

{
  "trans_id": "9b2bb872-332a-4d8d-87aa-2e18dc4edff8",
  "balance": 4000,
  "status": 1
}

Important:

Cancel request would be sent instantly if had not received status code 1.

Wallet API-Payout Request

Called when the User wins.
Operator is expected to increase member’s balance by amount and return new balance.

Before any altering of User’s balance, operator has to check that win wasn’t processed or canceled before.

Request URL

https://<url>/api/wallet/payout [POST]

Request Parameters

Parameter Type Enforce Description
trans_id String Yes transaction id PS:equal to transaction id requested
amount Number Yes transaction amount PS:support to second decimal place
agent String Yes agent account
account String Yes member account

Important:

operating transactionamount should be a positive number

Attention:

Retry Policy：
In case of network fail (HTTP 502, timeout, nxdomain, etc.) we will retry.
Until the bet may be rolled back, and the money returned back to user.
Every retry will extend its intermission.

Example

{
  "trans_id": "9b2bb872-332a-4d8d-87aa-2e18dc4edff8",
  "amount": 8000,
  "agent": "agent0001",
  "account": "player001",
}

Response Parameters

Parameter Type Description
trans_id String transaction id PS:equal to transaction id requested
balance Number transaction amount PS:support to second decimal place
status Number Wallet Message Code

Example

{
  "trans_id": "9b2bb872-332a-4d8d-87aa-2e18dc4edff8",
  "balance": 4000,
  "status": 1
}

Important:

Cancel request would be sent instantly if had not received status code 1.

Game API-Member

To Register Members in DS

To register a new member by API

Request URL

https://<url>/v1/member/create [POST]

Request Parameters

Parameter Type Enforce Description
channel String Yes channel(provided in MIF)
agent String Yes agent account
account String Yes member account

Example

{
  "channel": "53346874",
  "agent": "agent01",
  "account": "player001",
}

Response Parameters

Parameter Type Description
code Number wallet message code
message String status message

Example

{
  "result": {
    "code": 1,
    "msg": "success"
  }
}

Important:

The maximum text length of member account is 4~32 characters. The prefix is not required.
Characters should be:0~9, a~z, A~Z, -(hyphen), and _ (bottom line)

Get Bet Record URL for Details

To inquire the URL for bet details

Request URL

https://<url>/v1/record/get_bet_detail_page [POST]

Request Parameters

Parameter Type Enforce Description
game_serial String Yes serial for games
game_id String Yes game id
round_id String Yes round id
lang String Yes zh_cn and en_us supported

Example

{
  "game_id": "2001",
  "game_serial": "23134242",
  "round_id": "87122421352",
  "lang": "zh_cn"
}

Parameter Type Description
code Number status code
message String status message
url String bet record detail url 
{

    "code": 1,
